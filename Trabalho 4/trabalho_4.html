Trabalho 4 - Elaboração do Laboratório Experimental do SPV (Lex)

Roteiro

Bem-vindo ao teste do Sistema de Processamento Visual (SPV) para reconhecimento de placas veiculares. Este roteiro irá guiá-lo passo a passo para realizar o teste de maneira simples e eficiente.

Introdução

O objetivo do teste é verificar se o sistema SPV consegue identificar corretamente as placas veiculares cadastradas no banco de dados e rejeitar as que não estão registradas.

Preparação

Antes de começar, certifique-se de que o Tesseract está instalado na máquina. Caso esteja executando o programa pela primeira vez, abra o terminal e execute os comandos abaixo: 


$ sudo apt-get update
$ sudo apt-get install libsdl2-ttf-dev
$ sudo apt install libtesseract-dev

Princípio de funcionamento do programa e configurações iniciais

Abra a pasta placas_opencv no explorador de arquivo e, depois de aberta, abra o terminal e digite os comandos abaixo: 


$ sudo apt install libtesseract-dev 
$ cmake .
$ g++ -o placas_opencv placas_opencv.cpp `pkg-config --cflags --libs opencv4` -ltesseract `sdl2-config --cflags --libs` -lSDL2_ttf
$ ./placas_opencv

O código inicial está disposto abaixo:

#include <opencv2/opencv.hpp>
#include <tesseract/baseapi.h>
#include <iostream>
#include <chrono>
#include <ctime>
#include <sstream>
#include <filesystem>
#include <regex>

using namespace cv;
using namespace std;
namespace fs = std::filesystem;

// Função para aplicar processamento prévio à imagem
Mat preprocessImage(const Mat &img) {
    Mat gray, resized, dilated, eroded;

    // Redimensiona para aumentar a resolução
    resize(img, resized, Size(), 1.2, 1.2, INTER_CUBIC);

    // Converte para escala de cinza
    cvtColor(resized, gray, COLOR_BGR2GRAY);

    // Dilata e erode para destacar caracteres
    Mat kernel = getStructuringElement(MORPH_RECT, Size(1, 1));
    dilate(gray, dilated, kernel, Point(-1, -1), 1);
    erode(dilated, eroded, kernel, Point(-1, -1), 1);

    return eroded;
}

// Função para encontrar regiões candidatas
vector<Rect> detectRegions(const Mat &img) {
    Mat edges;
    vector<vector<Point>> contours;
    vector<Vec4i> hierarchy;
    

    // Detecção de bordas
    Canny(img, edges, 50, 200);

    // Encontrar contornos
    findContours(edges, contours, hierarchy, RETR_TREE, CHAIN_APPROX_SIMPLE);

    vector<Rect> candidateRegions;
    for (const auto &contour : contours) {
        Rect boundingBox = boundingRect(contour);

        // Filtrar regiões com base em proporção e tamanho
        double aspectRatio = (double)boundingBox.width / boundingBox.height;
        if (aspectRatio > 2.0 && aspectRatio < 6.0 && boundingBox.area() > 1000) {
            candidateRegions.push_back(boundingBox);
        }
    }
    return candidateRegions;
}

// Função para realizar OCR em uma região
string performOCR(const Mat &roi, tesseract::TessBaseAPI &ocrEngine) {
    Mat thresh;

    // Aplicar suavização e limiarização
    threshold(roi, thresh, 0, 255, THRESH_BINARY + THRESH_OTSU);

    // Realizar OCR
    ocrEngine.SetImage(thresh.data, thresh.cols, thresh.rows, 1, thresh.step);
    ocrEngine.SetSourceResolution(70);
    string text = string(ocrEngine.GetUTF8Text());

    // Limpar o texto detectado
    text = regex_replace(text, regex("[^A-Za-z0-9]"), "");
    transform(text.begin(), text.end(), text.begin(), ::toupper);

    return text;
}

int main() {
    // Banco de dados de placas cadastradas
    map<string, pair<string, string>> bancoDeDados = {
        {"PBA2019", {"Vinicius", "Honda Fit"}}
    };

    // Abrir a câmera (webcam)
    VideoCapture cap(0);
    if (!cap.isOpened()) {
        cerr << "Erro ao abrir a webcam!" << endl;
        return -1;
    }

    // Inicializar o OCR (Tesseract)
    tesseract::TessBaseAPI ocr;
    ocr.Init(NULL, "eng");

    cout << "Pressione a barra de espaço para verificar a placa detectada." << endl;
    cout << "Pressione 'q' para sair." << endl;

    while (true) {
        Mat frame;
        cap >> frame; // Captura um frame da webcam

        if (frame.empty()) {
            cerr << "Erro ao capturar o frame!" << endl;
            break;
        }

        // Mostrar o feed da câmera ao vivo
        imshow("Sistema de reconhecimento de placas veiculares", frame);

        char key = (char)waitKey(30);
        if (key == 'q' || key == 'Q') {
            break;
        } else if (key == ' ') { // Pressionou espaço para capturar
            // Pré-processamento da imagem
            Mat preprocessed = preprocessImage(frame);

            // Detectar regiões candidatas
            vector regions = detectRegions(preprocessed);

            for (const auto &region : regions) {
                Mat roi = preprocessed(region); // Recortar a região da imagem

                // Realizar OCR
                string placaTexto = performOCR(roi, ocr);

                // Validar texto detectado
                if (placaTexto.size() == 7) { // Apenas placas com 7 caracteres
                    cout << "Texto detectado: " << placaTexto << endl;

                    // Verificar no banco de dados
                    if (bancoDeDados.find(placaTexto) != bancoDeDados.end()) {
                        cout << "Acesso permitido!" << endl;
                        cout << "Motorista: " << bancoDeDados[placaTexto].first << endl;
                        cout << "Modelo do carro: " << bancoDeDados[placaTexto].second << endl;
                    } else {
                        cout << "Acesso negado! Placa não encontrada no sistema." << endl;
                    }
                }
            }
        }
    }

    // Finalizar OCR e liberar recursos
    ocr.End();
    cap.release();
    destroyAllWindows();

    cout << "Programa encerrado." << endl;
    return 0;
}

O programa utiliza uma série de técnicas de processamento de imagem para realizar o reconhecimento de placas veiculares. Essas técnicas são aplicadas para destacar características importantes da imagem e facilitar o processo de reconhecimento óptico de caracteres (OCR). Abaixo, detalhamos cada um dos métodos utilizados:

Redimensionamento da Imagem

  • Função: resize(img, resized, Size(), 1.2, 1.2, INTER_CUBIC)

  • Descrição: A imagem capturada é redimensionada para aumentar sua resolução. Isso ajuda a preservar os detalhes dos caracteres presentes na placa veicular.

  • Método: A interpolação bicúbica (INTER_CUBIC) é usada para melhorar a qualidade da imagem durante o redimensionamento.

Conversão para Escala de Cinza

  • Função: cvtColor(resized, gray, COLOR_BGR2GRAY)

  • Descrição: A imagem colorida é convertida para escala de cinza, reduzindo a complexidade e o ruído causado pelas cores. Isso é fundamental, pois o OCR funciona melhor em imagens monocromáticas.

  • Benefício: Reduz a quantidade de informações processadas, focando apenas nos tons de luminosidade.

Operações Morfológica

  • Funções: dilate(gray, dilated, kernel, Point(-1, -1), 1) e erode(dilated, eroded, kernel, Point(-1, -1), 1)

  • Descrição: A dilatação expande as áreas claras da imagem, conectando partes que possam estar quebradas. E, a erosão remove ruídos pequenos e ajusta os contornos das regiões dilatadas.

  • Kernel Utilizado: Uma matriz retangular (MORPH_RECT) de tamanho 1x1, que é suficiente para processar caracteres finos como os encontrados em placas veiculares.

Detecção de Bordas

  • Função: Canny(img, edges, 50, 200)

  • Descrição: O método de Canny é utilizado para detectar bordas na imagem. Ele identifica mudanças bruscas de intensidade, como as que ocorrem nos limites dos caracteres.

  • Parâmetros: 50 e 200 - Limiar inferior e superior para a detecção de bordas. Esses valores foram escolhidos para equilibrar a sensibilidade do algoritmo.

Detecção de Contornos

  • Função: findContours(edges, contours, hierarchy, RETR_TREE, CHAIN_APPROX_SIMPLE)

  • Descrição: Após a detecção de bordas, os contornos são encontrados. Cada contorno representa uma possível região de interesse que pode conter a placa veicular.

  • Método: O método RETR_TREE garante que a hierarquia dos contornos seja preservada, permitindo identificar relações entre eles. E, o CHAIN_APPROX_SIMPLE reduz o número de pontos armazenados, otimizando o processamento.

Filtragem de Regiões

  • Função: if (aspectRatio > 2.0 && aspectRatio < 6.0 && boundingBox.area() > 1000)

  • Descrição: Após identificar as regiões candidatas, elas são filtradas com base em:

    • Proporção (Aspect Ratio): Apenas regiões que têm um formato retangular semelhante ao de uma placa veicular (comprimento maior que a altura) são consideradas.

    • Área: Regiões muito pequenas, que provavelmente não contêm informações úteis, são descartadas.

imiarização (Thresholding)

  • Função: threshold(roi, thresh, 0, 255, THRESH_BINARY + THRESH_OTSU)

  • Descrição: A região de interesse (ROI) é convertida para uma imagem binária.

  • Método de Otsu: Determina automaticamente o melhor limiar para separar os caracteres do fundo, maximizando o contraste.

  • Benefício: Facilita a detecção dos caracteres pelo OCR, eliminando variações de intensidade.
