Laboratório 1 - Captura de Imagem e Vídeo

Introdução

O presente laboratório tem como objetivo realizar análises gráficas a partir da captura e do processamento de imagens e vídeos utilizando a biblioteca OpenCV. Neste sentido, o OpenCV se destaca como uma ferramenta poderosa para o processamento de imagens e vídeos, permitindo não apenas a manipulação de arquivos multimídia, mas também a interação com dispositivos de captura, como webcams, além de possibilitar a realização de transformações e análises em tempo real.


Fundamentos básicos

O processamento de imagens e vídeos é um campo fundamental da visão computacional, responsável pela análise e manipulação de dados visuais. A biblioteca OpenCV (Open Source Computer Vision Library) é uma das ferramentas mais utilizadas nessa área, oferecendo uma ampla gama de recursos para capturar, processar e exibir imagens e vídeos. Com suporte a diversas linguagens de programação, como Python e C++, o OpenCV possibilita a implementação de algoritmos de visão computacional de maneira eficiente e em tempo real.

A captura de imagens e vídeos pode ser realizada tanto a partir de arquivos pré-existentes quanto por meio de dispositivos como webcams. Uma vez capturados, esses dados visuais podem ser processados para diferentes finalidades, como detecção de objetos, rastreamento de movimento ou reconhecimento facial. O processo envolve uma série de operações que convertem os dados visuais brutos em informações úteis e significativas.

Programa 1 – video_read_from_file:

O programa lê um arquivo de extensão .mp4, verifica se o arquivo foi carregado corretamente e em seguida reproduz frame a frame em um loop até que todos os frames tenham sido reproduzidos. Esse processo segue em repetição até que o usuário pressione a tecla “Q”. Alterando os valores de definidos para “k” (que recebe waitKey), conseguimos visualizar os frames individualmente. O programa foi utilizado apenas para estudo e não realizamos alterações. A utilidade desse programa seria para um caso onde já se tem o arquivo de vídeo disponível para a leitura.

Assim, realizou-se a execução deste programa para o vídeo abaixo: 


// Include Libraries
#include <opencv2/opencv.hpp>
#include <iostream>

// Namespace to nullify use of cv::function(); syntax
using namespace std;
using namespace cv;

int main()
{
  // initialize a video capture object
  VideoCapture vid_capture("Lab 1/big_buck_bunny.mp4");

  // Print error message if the stream is invalid
  if (!vid_capture.isOpened())
  {
  	cout << "Error opening video stream or file" << endl;
  }

  else
  {
    // Obtain fps and frame count by get() method and print
    // You can replace 5 with CAP_PROP_FPS as well, they are enumerations
    int fps = vid_capture.get(5);
    cout << "Frames per second :" << fps;

    // Obtain frame_count using opencv built in frame count reading method
    // You can replace 7 with CAP_PROP_FRAME_COUNT as well, they are enumerations
    int frame_count = vid_capture.get(7);
    cout << "  Frame count :" << frame_count;
  }


  // Read the frames to the last frame
  while (vid_capture.isOpened())
  {
    // Initialise frame matrix
    Mat frame;

    // Initialize a boolean to check if frames are there or not
    bool isSuccess = vid_capture.read(frame);

    // If frames are present, show it
    if(isSuccess == true)
    {
      //display frames
      imshow("Frame", frame);
    }

    // If frames are not there, close it
    if (isSuccess == false)
    {
      cout << "Video camera is disconnected" << endl;
      break;
    }

    //wait 20 ms between successive frames and break the loop if key q is pressed
    int key = waitKey(20);
    if (key == 'q')
    {
      cout << "q key is pressed by the user. Stopping the video" << endl;
      break;
    }

  }
  // Release the video capture object
  vid_capture.release();
  destroyAllWindows();
  return 0;
}

Programa 2 – video_read_from_image_sequence:

Consiste no mesmo funcionamento do programa 1, porém, a leitura se dá através de uma sequência de imagens no repositório especificado. O programa reproduz todas as 200 imagens e ao fim encerra. Também é possível antecipar o encerramento do programa apertando a tecla “Q” assim como no programa anterior. Utilizamos o programa apenas para estudo, a única alteração necessária foi do caminho até o repositório das imagens.

Esse programa é útil quando somente estão disponíveis os frames individuais de uma cena e busca-se visualizar como vídeo ou então quando o objetivo é fazer uma apresentação de imagens.

Programa 3 – video_read_from_webcam:

Esse programa faz a leitura de imagens diretamente obtidas pela webcam e em seguida reproduz em uma janela. Assim como os programas descritos acima, o código possui trechos de validação para garantir que a captura está ocorrendo corretamente e as imagens estão sendo localizadas. O encerramento da execução se dá também ao pressionar a tecla “Q”. Também é possível modificar o intervalo de tempo entre frames causando um "delay" na reprodução da webcam. Esse programa é útil para obter imagens de uma webcam em tempo real como em softwares e sites de videochamada.

Programa 4 – video_write_from_webcam:

Este programa faz a leitura da imagem obtida com uma webcam e grava essa reprodução em um arquivo após o encerramento da execução do código ao pressionar a tecla "Q". É possível fazer alterações no programa para mudar o tipo de extensão de saída do arquivo de vídeo, assim como manipular o espaçamento entre frames causando o efeito de gargalo entre ‘cenas’ . Um uso direto desse programa seria em uma gravação, por exemplo, para vídeo aulas em que um professor utiliza sua webcam e pretende disponibilizar a gravação para que os alunos consultem posteriormente. Uma outra aplicação seria o uso para gravações no geral.

Programa 5 – video_write_from_webcam:

Este programa, diferentemente do anterior que executa a webcam e grava o que está sendo capturado com a webcam, reproduz um arquivo de vídeo já salvo em um diretório especificado e grava esse vídeo em reprodução. Dessa forma, uma aplicação para esse programa poderia ser a de um conversor de formatos de vídeo. Partindo de um vídeo no formato .mp4 pode-se obter um vídeo com extensão .AVI. Esse programa poderia ser usado para automatizar uma conversão de um vídeo que foi gravado em um formato específico e adequá-lo para reprodução em um outro sistema que suporta apenas extensões específicas.
